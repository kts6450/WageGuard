# CLAUDE.md — WageGuard 프로젝트 가이드

이 문서는 이 저장소에서 작업하는 AI 어시스턴트(Claude Code)와 협업자를 위한 가이드다.

## 프로젝트 한 줄 요약

승인·로그인이 불필요한 **전면 공개 행정데이터**만으로 임금 착취 위험을 시도·시군구×업종 **셀 단위로 스크리닝**하고, 근로감독 우선순위를 제안하는 시스템. **개별 사업장을 지목하지 않는다.**

## 핵심 명령어

```bash
python scripts/refresh_all.py              # 월별 전체 갱신 (수집→재계산→이력 적재)
python scripts/refresh_all.py --skip-fetch # 재계산만

streamlit run dashboard/ops_app.py         # 감독관용 운영 앱 (실사용)
streamlit run dashboard/screening_app.py   # 검증·시연 앱 (발표용)

python scripts/build_presentation.py       # WageGuard_발표.pptx 재생성
python scripts/build_proposal_pdf.py       # PROPOSAL.html/pdf 재생성
python scripts/audit_validation.py         # 검증 수치 적대적 감사
python scripts/audit_deck_layout.py        # PPT 텍스트 오버플로 감사
```

## 절대 지켜야 할 원칙

1. **순환논리 차단 구조 유지**: 위험 지표는 국민연금 데이터만으로 산출, 체불 명단은 채점에만 사용. 이 분리를 깨는 변경 금지.
2. **수치 출처 단일화**: 발표·문서에 인용하는 모든 수치는 `results/cell_screening.json`과 `results/validation_audit.json`이 원본. 임의 수치 작성 금지.
3. **효율 배수는 두 기준 병기**: 고용(가입자) 기준 5.2배 · 사업장 수 기준 2.4배 — 하나만 쓰지 않는다.
4. **"홀드아웃 차수"라고 표현** ("미래 예측" 금지 — 국민연금 스냅샷이 명단 공개보다 나중 시점).
5. **폐기된 구 수치 사용 금지**: AUROC 0.9998(라벨 누수로 자진 철회), "E9 66% 계약 미인지"(집계 오류로 철회), 이주노동자 90만(→ 통계청 2025년 110.9만), 체불 명단 789명(→ 중복 제거 769명), 점검 효율 7.2배(→ 실측 5.2/2.4배).
6. **국민연금 업종코드는 KSIC 8차(구분류)**: 건설 45-46, 제조 15-37, 도소매 50-52. 10차 기준으로 매핑하면 건설업이 통째로 누락된다.
7. **가점 데이터 유지**: 한국고용정보원(주관기관) 외국인근로자 근무현황·EIS 연계는 어떤 수정에서도 삭제하지 않는다 (공모 가점 '나' 2점).
8. 산출은 셀 단위로 한정 — 개별 사업장 지목·자동 처분 기능을 추가하지 않는다.
